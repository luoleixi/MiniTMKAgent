# MiniTMK Agent 架构设计文档

## 1. 系统概述

MiniTMK Agent 是一个实时同声传译系统，支持语音输入、实时翻译和语音输出。

### 1.1 核心功能
- **实时语音识别 (ASR)**: 将语音转换为文本
- **机器翻译 (MT)**: 将源语言文本翻译为目标语言
- **语音合成 (TTS)**: 将翻译后的文本转换为语音
- **音频播放**: 按序播放合成的语音

### 1.2 系统架构图

```
┌─────────────┐     ┌─────────────────────────────┐     ┌─────────────┐
│   麦克风    │────▶│  百炼平台 (Paraformer ASR)  │────▶│  翻译引擎   │
│  (输入)     │     │  - 内置VAD自动断句          │     │  (百炼平台) │
└─────────────┘     │  - 实时流式识别             │     └──────┬──────┘
                    └─────────────────────────────┘            │
                                                               │
                              ┌────────────────────────────────┘
                              ▼
                       ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
                       │    TTS      │────▶│  播放队列   │────▶│   扬声器    │
                       │ (语音合成)   │     │ (顺序播放)   │     │  (输出)     │
                       └─────────────┘     └─────────────┘     └─────────────┘
```

## 2. 模块设计

### 2.1 音频模块 (internal/audio)

#### 2.1.1 录音器 (Recorder)
- **功能**: 从麦克风捕获音频
- **实现**: 使用 `miniaudio` (malgo) 库
- **采样率**: 16kHz (与百炼平台匹配)
- **缓冲区**: 累积0.5秒音频后批量发送

#### 2.1.2 播放器 (Player)
- **功能**: 播放 PCM 音频
- **实现**: 使用 `miniaudio` (malgo) 库

#### 2.1.3 播放队列 (PlayQueue)
- **功能**: 确保多条语音按顺序播放
- **机制**:
  - 序列号生成器分配递增 ID
  - 乱序到达的音频存入 pending 队列
  - 按序列号顺序播放

### 2.2 TTS 模块 (internal/tts)

#### 2.2.1 接口设计
```go
type TTS interface {
    Synthesize(text, voice string) ([]byte, error)
    SynthesizeStream(text, voice string, onAudioChunk func(chunk []byte)) error
    Close() error
}
```

#### 2.2.2 百炼平台实现 (BaiwanTTS)
- **协议**: WebSocket 流式 API
- **模型**: CosyVoice-v3 (多语言通用)
- **交互流程**:
  1. `run-task` → 启动任务
  2. `task-started` ← 任务就绪
  3. `continue-task` → 发送文本
  4. `finish-task` → 结束输入
  5. `task-finished` ← 任务完成

- **多语言支持**: 使用 CosyVoice-v3 模型，支持跨语言合成

#### 2.2.3 连接管理
- 每个 TTS 请求使用独立的 WebSocket 连接
- 避免并发请求导致连接混乱

### 2.3 翻译模块 (internal/translator)

#### 2.3.1 百炼平台翻译
- **API**: 阿里云百炼平台 (qwen-turbo 模型)
- **特点**: 针对口语优化的翻译质量

### 2.4 识别模块 (internal/recognizer)

#### 2.4.1 百炼平台 ASR
- **API**: 阿里云百炼平台实时语音识别
- **模型**: paraformer-realtime-v2
- **协议**: WebSocket 流式
- **内置VAD**: 平台自动检测语音活动和断句
- **支持**: 中文、英文、日文等多种语言

### 2.5 Agent 模块 (internal/agent)

#### 2.5.1 StreamAgent
- **职责**: 协调录音、识别、翻译、TTS、播放全流程
- **特点**:
  - 序列号管理确保播放顺序
  - 异步 TTS 避免阻塞识别
  - 超时控制防止无限等待

#### 2.5.2 配置管理
- **存储位置**:
  - Windows: `%APPDATA%\mini-tmk-agent\config.json`
  - macOS: `~/Library/Application Support/mini-tmk-agent/config.json`
  - Linux: `~/.config/mini-tmk-agent/config.json`

## 3. 数据流

### 3.1 实时同传流程

```
1. 用户说话 → 录音器捕获音频
2. 批量发送 → 累积0.5秒后发送到百炼平台
3. ASR识别 → 平台内置VAD自动断句，返回文本
4. 翻译引擎 → 获取目标语言文本
5. 分配序列号 → 进入 TTS 队列
6. TTS 合成 → 获取音频数据
7. 播放队列 → 按序播放
```

### 3.2 序列号机制

```
句子1 (长文本) → 序列号 #1 → TTS (慢)
句子2 (短文本) → 序列号 #2 → TTS (快)
                           ↓
              播放队列: #2 先完成，等待 #1
                           ↓
              #1 完成后播放 → 然后播放 #2
```

## 4. 并发设计

### 4.1 协程模型

```
主协程
  ├── 录音协程 (持续读取音频)
  ├── 流式发送 (累积后批量发送)
  ├── 识别协程 (发送/接收 ASR 结果)
  └── 播放协程 (后台播放队列)

每个 TTS 请求
  └── 独立的 WebSocket 连接协程
```

### 4.2 同步机制

- **互斥锁 (sync.Mutex)**: 保护共享状态
- **通道 (chan)**: 协程间通信
- **Context**: 取消信号传播

## 5. 错误处理

### 5.1 错误分类

| 类型 | 处理策略 |
|------|----------|
| 网络错误 | 重试/超时后报错 |
| API 错误 | 打印错误，继续运行 |
| 音频错误 | 跳过当前片段 |
| 配置错误 | 首次启动引导配置 |

### 5.2 超时控制

- TTS 合成: 10 秒超时
- WebSocket 读取: 30 秒超时
- 连接建立: 10 秒超时

## 10. 性能指标

- **端到端延迟**: 1-3 秒 (取决于网络)
- **并发 TTS**: 无限制 (每个请求独立连接)
- **音频质量**: 16kHz, 16bit, 单声道
- **支持语言**: 8 种 (zh, en, ja, ko, fr, de, es, ru)
